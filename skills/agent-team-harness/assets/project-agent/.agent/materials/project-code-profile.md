## Current Style Facts

- Use existing package, module, naming, error handling, and test style before
  introducing new patterns.
- Keep public contracts explicit and compatible unless a breaking change is
  approved.
- External calls need explicit timeout, retry, and failure behavior.
- Shared mutable state requires a documented concurrency strategy.
- Boundary code may catch broad exceptions only when it normalizes downstream
  failure or protects best-effort observability.

## Stability Invariants

- Invalid requests must not reach downstream side-effecting operations.
- State transitions must be idempotent or guarded by an explicit duplicate
  handling strategy.
- Cache, MQ, DB, RPC, and third-party calls need failure and rollback behavior.
- Observability must show caller-visible failures, degradation, and retries.
